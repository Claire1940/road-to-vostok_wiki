## Metro Mod Loaderとは？

コミュニティメンバーのMetroによって開発された **metro mod loader** は、Road to VostokのGodotベースアーキテクチャ向けに特化して設計された軽量スクリプトローディングユーティリティです。コアゲームデータを上書きしがちな従来のMODマネージャーとは異なり、このローダーはオーバーライドシステムを利用して、ゲーム初期化フェーズ中にカスタムスクリプトやアセットを注入します。

このツールは、ゲーム更新が頻繁に行われる2026年の早期アクセス期間において特に重要です。ローダー自体は高い耐性を持つよう設計されていますが、プレイヤーは常に現在の「EA」リリースと互換性のあるバージョンを使用していることを確認してください。

| Component | Description | Primary Function |
| :--- | :--- | :--- |
| **mod_loader.gd** | コアスクリプトファイル | フォルダ内のすべてのMODの読み込み順制御と注入を処理します。 |
| **override.cfg** | 設定ファイル | ゲームエンジンがMOD化されたスクリプトを認識するよう指示します。 |
| **Mods Folder** | ユーザー作成ディレクトリ | すべての `.pck` またはスクリプトファイルを保存する中心的な保管場所です。 |

## インストール前準備：データのバックアップ

ゲームファイルを変更する前に、既存の進行データを保護するのはベストプラクティスです。Road to Vostokは、重要なセーブデータや設定ファイルをローカルのAppDataディレクトリに保存しています。MOD導入は通常安全ですが、MODの不適切な削除やバージョン競合によって、まれにセーブ破損が発生する可能性があります。

1.  キーボードで `Windows + R` を押し、ファイル名を指定して実行ダイアログを開きます。
2.  `%appdata%` と入力してEnterを押します。
3.  `RoadToVostok` フォルダに移動します（古いデモ用フォルダではなく、現行バージョンであることを確認してください）。
4.  **Backup_2026** という名前の新規フォルダを作成します。
5.  `.tre` で終わるすべてのファイルと設定ファイルをこのバックアップフォルダにコピーします。

> **Warning:** ゲームファイルの変更は予期しないクラッシュを引き起こす可能性があります。新しいバージョンのmod loaderをインストールする前に、必ず最新のセーブバックアップを保持してください。

## ステップ別インストールガイド

**metro mod loader** のインストールでは、特定のファイルを2つの別々の場所（SteamのゲームディレクトリとAppDataシステムフォルダ）へ移動します。ローダーが正しく初期化されるよう、以下の手順を慎重に実行してください。

### Part 1: The Game Directory
まず、ローカルゲームファイルの場所を確認します。Steamで *Road to Vostok* を右クリックし、**Manage**、次に **Browse local files** を選択してください。

| File to Move | Destination | Action |
| :--- | :--- | :--- |
| **override.cfg** | ゲームルートフォルダ | 求められた場合は上書き（更新時のみ）。 |
| **New Folder** | ゲームルートフォルダ | **mods**（すべて小文字）という名前のフォルダを作成。 |

### Part 2: The AppData Directory
ローダーの残り半分は、ゲームが永続データを保存する場所に配置します。

1.  バックアップ手順で開いた `%appdata%/RoadToVostok` フォルダに戻ります。
2.  **mod_loader.gd** ファイルをこのディレクトリ直下に移動します。
3.  ファイルをサブフォルダ内に置かないでください。エンジンに検出されるには、セーブファイルと同じ階層に置く必要があります。

## Mod Configuration Menu（MCM）の統合

**metro mod loader** が有効になったら、次に欠かせないのがMod Configuration Menu、通称MCMです。コミュニティ開発者 doink oink によって作成されたこの「MODのためのMOD」は、ゲーム設定内にグラフィカルユーザーインターフェースを提供します。

MCMがない場合、多くの複雑なMODでは、FOV・AI難易度・スポーン率などの設定変更にテキストファイルの手動編集が必要になります。MCMを導入すれば、これらの項目をメインメニュー内のスライダーやトグルで操作できるようになります。

### How to Install MCM:
1.  2026 EAリリースに対応したMCMパッケージをダウンロードします（紫色の互換タグが目印）。
2.  ファイルを展開し、生成されたフォルダ/ファイルをゲームディレクトリ内に作成した `mods` フォルダへ配置します。
3.  ゲームを起動します。通常の設定メニュー内に新しい「MCM」ボタンが表示されるはずです。

## ロード順と優先度の管理

ローダー導入後にRoad to Vostokを起動すると、通常はメインメニューの前にポップアップウィンドウが表示されます。この画面では、個別MODの有効化/無効化に加えて、より重要な優先度設定が可能です。

優先度システムは数値で動作し、**数字が小さいほど先に読み込まれます**。安定した環境のため、次の優先度構成を推奨します。

| Mod Type | Recommended Priority | Reason |
| :--- | :--- | :--- |
| **MCM (Mod Config Menu)** | 0 または 1 | 他MODが設定を登録できるよう、最初に読み込む必要があります。 |
| **Core Frameworks** | 5-10 | 他MOD向けアセットを提供する種類のMOD。 |
| **Overhauls (e.g., Immersive Overhaul)** | 20 | 大規模変更はフレームワーク確立後に読み込むべきです。 |
| **Small Tweaks** | 50+ | 個別武器スキンやサウンド差し替えは最後で問題ありません。 |

> **Tip:** 起動時にクラッシュする場合は、ローダー画面ですべてのMODをいったん外し、1つずつ再有効化して原因を特定してください。

## よくある問題のトラブルシューティング

**metro mod loader** のような堅牢なツールでも、バージョン不一致やファイル配置ミスによって問題が起こる場合があります。

*   **Loader Not Appearing:** `override.cfg` がルートフォルダにあり、`mod_loader.gd` がAppDataフォルダにあることを確認してください。どちらかが欠けていると、ゲームはエラーなしで「バニラ」モード起動になります。
*   **MCM Button Missing:** 通常、MCM MODが `mods` ディレクトリ内の深すぎるサブフォルダに置かれていると発生します。フォルダ構造は浅く保ってください。
*   **Performance Drops:** MCMでAI数を増やすと、CPU性能に大きく影響する可能性があります。カクつく場合はAIスポーン上限を10以下に下げてください。

## FAQ

**Q: metro mod loaderは無料デモ版でも動作しますか？**

A: デモ向けローダーのバージョンは存在しましたが、現行版はEarly Accessリリース向けに最適化されています。デモ版とEA版ではファイル構造が大きく異なるため、クラッシュ回避のためにもEA互換版の **metro mod loader** を使用することを強く推奨します。

**Q: このローダーでマルチプレイはできますか？**

A: 現在、Road to Vostokはシングルプレイ中心の体験です。MOD導入はローカルプレイを想定しています。将来マルチプレイ機能が追加された場合、同期ズレ防止のため、全プレイヤーがまったく同じMODリストとロード順を使う必要が出てくる可能性が高いです。

**Q: すべてのMODを完全にアンインストールするには？**

A: 完全なバニラ状態に戻すには、ゲームディレクトリ内の `mods` フォルダを削除し、`override.cfg` ファイルを取り除いてください。さらに、AppDataフォルダから `mod_loader.gd` ファイルも削除します。これでゲームはカスタムスクリプトの読み込みを試行しなくなります。

**Q: クラッシュが止まらない場合、どこで助けを得られますか？**

A: 最適な情報源はRoad to Vostok Modding Discordです。ローダーや各種オーバーホールパッケージをサポートする専用の「user-help」チャンネルがあります。質問時は必ずクラッシュログを添えてください。
