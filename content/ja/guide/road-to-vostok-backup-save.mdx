データを保護する前に、ゲームが「肝心な部分」をどこに保存しているかを知る必要があります。Road to Vostok は、実験的なデータのすべてに標準のSteam Cloudを使用しているわけではないため、手動での介入が必要です。

**ビデオのハイライト:**
- AppDataディレクトリ内の正しいフォルダを特定する。
- 早期アクセスバージョンと古いデモファイルの違いを区別する。
- 正常な復元に必要な特定の .TRES ファイルを選択する。
- 整理されたセーブ管理のための専用バックアップフォルダを作成する。

| ファイルタイプ | 重要度 | 説明 |

<Section icon={Shield} title="手動バックアッププロセス">

**Road to Vostok のバックアップセーブ**の作成は、各ゲームセッションまたはModのインストール前に行うべき簡単なプロセスです。データが正しく保存されていることを確認するために、以下の手順に従ってください。

<Steps>
  <Step num="1" title="「ファイル名を指定して実行」コマンドを開く">
    キーボードの **Windowsキー + R** を押して、「ファイル名を指定して実行」ダイアログボックスを開きます。**%appdata%** と入力し、Enterキーを押します。これにより、Roamingフォルダに直接移動します。
  </Step>
  <Step num="2" title="Road to Vostok に移動する">
    **Road to Vostok** という名前のフォルダを見つけます。それを開くと、シェーダーフォルダや設定ファイルを含む内部セーブコンポーネントが表示されます。
  </Step>
  <Step num="3" title="コアデータをコピーする">
    すべての **.TRES** ファイルを強調表示します。フォルダ全体をコピーすることもできますが、これらの特定のファイルが最も重要です。右クリックして **コピー** を選択します。
  </Step>
  <Step num="4" title="安全なストレージを作成する">
    デスクトップまたはクラウドドライブに **Vostok Backups** という名前の新しいフォルダを作成します。その中に、現在の日付（例: 2026-05-16）のサブフォルダを作成し、そこにファイルを貼り付けます。
  </Step>
</Steps>

<Section icon={Sparkles} title="安全なMod導入と統合">

2026年におけるMod導入はMetro Mod Loaderによってより効率化されましたが、それでも慎重なファイル管理が必要です。Real Gun Names や Mod Configuration Menu (MCM) のようなModを使用する予定がある場合、バックアップセーブがあなたのセーフティネットとなります。

| Modコンポーネント | インストールパス | 目的 |
| :--- | :--- | :--- |
| Metro Mod Loader | ゲームのルートフォルダ | 外部VMZファイルをロードするために必要なコアエンジンです。 |
| Override Config | ゲームのルートフォルダ | ローダーがMod化されたアセットをバニラよりも優先することを可能にします。 |
| Mods Folder | Root/mods | すべての.VMZ Modが格納される手動で作成されたフォルダです。 |
| VMZ Files | Root/mods | テクスチャ、スクリプト、またはデータを含む個々のModファイルです。 |

Modを適切にインストールするには、まずSteam経由でローカルゲームファイルを見つける必要があります。ライブラリでRoad to Vostokを右クリックし、**管理**を選択し、次に**ローカルファイルを閲覧**を選択します。ここにModローダーを配置し、**mods**フォルダを作成します。

<Callout type="info" title="MCMの重要性">
**Mod Configuration Menu (MCM)** は、常に最初にインストールすべきModです。反動スライダーやアタッチメントの切り替えなど、他のMod内の設定を調整するための必要なユーザーインターフェースを提供します。
</Callout>

</Section>
